# 이메일 기반 회원가입

<p>사용자가 이메일, 비밀번호, 닉네임을 사용하여 서비스에 새로운 계정을 생성할 수 있도록 합니다.</p>

<h3>Objective</h3>
<p>신규 사용자가 최소한의 정보 입력으로 쉽고 빠르게 서비스에 가입할 수 있도록 하여, 가입 전환율을 높이고 초기 사용자 이탈을 방지합니다.</p>

<h3>Key Results</h3>
<ul>
    <li>회원가입 절차를 1분 이내에 완료할 수 있다.</li>
    <li>입력 데이터 유효성 검사를 통해 잘못된 정보로 인한 가입 실패율을 5% 미만으로 유지한다.</li>
    <li>가입 성공 시 사용자에게 즉각적인 피드백을 제공하여 0.5초 내에 다음 단계(로그인 페이지)로 안내한다.</li>
</ul>

<h3>기능 명세</h3>
<ul>
    <li><b>가입 UI:</b> 이메일, 비밀번호, 닉네임 입력 필드와 '회원가입' 버튼을 제공한다.</li>
    <li>
        <b>API Endpoint:</b> <code>POST /api/register</code>
    </li>
    <li>
        <b>Request Body:</b>
        <pre><code>{
  "email": "user@example.com",
  "password": "password123",
  "nickname": "testuser"
}</code></pre>
    </li>
    <li><b>유효성 검사 (Validation):</b>
        <ul>
            <li>모든 필드(<code>email</code>, <code>password</code>, <code>nickname</code>)는 필수 항목이다.</li>
            <li>이메일은 유효한 형식이어야 한다.</li>
            <li>비밀번호는 최소 6자 이상이어야 한다.</li>
            <li>닉네임은 최소 3자 이상이어야 한다.</li>
            <li>서버는 이메일과 닉네임이 시스템 내에서 고유한지 확인한다.</li>
        </ul>
    </li>
    <li><b>프로세스:</b>
        <ol>
            <li>사용자가 정보를 입력하고 '회원가입' 버튼을 클릭하면, 데이터는 서버로 전송된다.</li>
            <li>서버는 입력값에 대한 유효성 검사를 수행한다.</li>
            <li>유효성 검사를 통과하면, 비밀번호를 안전하게 해싱(bcrypt)하여 데이터베이스에 저장한다.</li>
            <li>새로운 사용자 레코드를 생성한다.</li>
        </ol>
    </li>
    <li><b>응답 (Responses):</b>
        <ul>
            <li><b>성공 (201 Created):</b> 회원가입이 성공하면, 서버는 생성된 사용자의 정보(비밀번호 제외)를 포함한 JSON 객체로 응답한다.
            <pre><code>{
  "message": "User created successfully!",
  "user": {
    "id": "60d0fe4f5311236168a109ca",
    "email": "user@example.com",
    "nickname": "testuser"
  }
}</code></pre>
            </li>
            <li><b>오류 - 잘못된 요청 (400 Bad Request):</b> 이메일 형식이 잘못되었거나, 비밀번호/닉네임 길이가 짧은 경우 등 유효성 검사에 실패했을 때 응답한다.
            <pre><code>{
  "message": "Validation failed.",
  "errors": [
    {
      "type": "field",
      "value": "pass",
      "msg": "Password must be at least 6 characters long.",
      "path": "password",
      "location": "body"
    }
  ]
}</code></pre>
            </li>
            <li><b>오류 - 충돌 (409 Conflict):</b> 이미 사용 중인 이메일이나 닉네임으로 가입을 시도할 경우 응답한다.
            <pre><code>{
  "message": "E-mail address already exists!"
}</code></pre>
            </li>
        </ul>
    </li>
</ul>
