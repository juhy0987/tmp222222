# 로그인/로그아웃 기능

<h3>Objective</h3><p>사용자가 자신의 계정으로 서비스에 안전하고 신속하게 접근하고, 사용 종료 시 명확하게 세션을 종료할 수 있도록 한다.</p><h3>Key Results</h3><ul><li>로그인 API 평균 응답 시간 500ms 미만</li><li>로그인 실패 시 명확한 오류 메시지 제공으로 인한 재시도 성공률 20% 증가</li><li>로그아웃 후 세션 정보 즉시 만료 및 로그인 페이지로 0.5초 내 리디렉션</li></ul><h3>기능 명세</h3><ul><li><b>로그인 UI:</b> 이메일과 비밀번호 입력 필드를 제공하며, '자동 로그인' 체크박스를 기본으로 선택된 상태로 둔다. 비밀번호 필드에는 '비밀번호 보기' 아이콘을 제공하여 사용자가 입력한 내용을 확인할 수 있도록 한다.</li><li><b>인증 프로세스:</b> 사용자가 '로그인' 버튼을 클릭하면, 입력된 정보는 HTTPS를 통해 서버로 전송된다. 서버는 계정 정보 일치 여부를 확인하고, 성공 시 JWT(Access Token, Refresh Token)를 발급한다. Access Token은 1시간, Refresh Token은 14일의 유효기간을 가진다. 클라이언트는 발급받은 토큰을 안전하게 저장(HTTPOnly 쿠키 또는 로컬 스토리지)하고, 이후 모든 API 요청 헤더에 Access Token을 포함하여 전송한다.</li><li><b>오류 처리:</b> 로그인 실패 시, "이메일 또는 비밀번호가 일치하지 않습니다." 또는 "존재하지 않는 계정입니다." 와 같이 구체적인 오류 메시지를 입력 필드 하단에 0.2초 내에 표시한다. 5회 연속 로그인 실패 시, 계정을 10분간 잠금 처리하고 사용자에게 해당 사실을 알린다.</li><li><b>로그아웃:</b> 사용자가 헤더의 프로필 메뉴에서 '로그아웃'을 클릭하면, 클라이언트에 저장된 토큰 정보를 즉시 삭제하고 서버에 로그아웃 API를 호출하여 서버 측 세션도 무효화한다. 이후 즉시 로그인 페이지로 리디렉션한다.</li></ul>
