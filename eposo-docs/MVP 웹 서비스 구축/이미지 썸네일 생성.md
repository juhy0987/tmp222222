# 이미지 썸네일 생성

<h3>Objective</h3><p>사용자가 파일 목록에서 이미지 파일을 이름만으로 구분하는 것이 아니라, 시각적인 축소판 이미지를 통해 직관적으로 내용을 파악할 수 있도록 한다.</p><h3>Key Results</h3><ul><li>이미지 파일 업로드 후 썸네일 생성 및 표시까지 평균 3초 이내 (5MB 이미지 기준)</li><li>썸네일 로딩으로 인한 파일 목록 UI 렌더링 지연 시간 200ms 미만</li><li>생성된 썸네일과 원본 이미지의 내용 일치율 100%</li></ul><h3>기능 명세</h3><ul><li><b>썸네일 생성 프로세스:</b> 사용자가 이미지 파일(JPG, PNG, GIF 등 지원 형식)을 업로드하면, 서버는 업로드 완료 직후 비동기적으로 썸네일 생성 작업을 시작한다. 원본 이미지의 가로세로 비율을 유지하면서, 긴 쪽을 기준으로 256x256 픽셀 크기의 썸네일을 생성한다. 생성된 썸네일은 별도의 경로에 캐싱되어 추후 빠른 로딩을 지원한다.</li><li><b>UI 표시:</b> 파일 목록이 그리드 뷰(Grid View) 형식일 때, 각 이미지 파일은 이름 대신 생성된 썸네일로 표시된다. 리스트 뷰(List View)에서는 파일명 좌측의 파일 타입 아이콘 대신 작은 크기(예: 40x40 픽셀)의 썸네일을 표시한다. 썸네일이 아직 생성 중일 때는 기본 이미지 아이콘 또는 스켈레톤 UI를 보여주다가, 생성이 완료되면 자연스럽게 썸네일 이미지로 전환된다. 썸네일 이미지는 레이지 로딩(Lazy Loading)을 적용하여 현재 화면에 보이는 부분만 로드하여 초기 로딩 속도를 최적화한다.</li></ul>
