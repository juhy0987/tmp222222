# 파일명 기반 검색 기능

<h3>Objective</h3><p>사용자가 저장된 수많은 파일 중에서 원하는 파일을 빠르고 정확하게 찾을 수 있도록 한다.</p><h3>Key Results</h3><ul><li>100만개 파일 기준, 검색 결과 평균 응답 시간 1초 미만 달성</li><li>검색 정확도 99% 이상 (검색어 포함 파일 누락 없음)</li><li>검색 기능 사용 후 실제 파일 클릭 전환율 80% 이상</li></ul><h3>기능 명세</h3><ul><li><b>UI/UX:</b> 웹 페이지 헤더의 우측 상단에 돋보기 아이콘과 함께 검색창을 고정 배치한다. 사용자가 검색창에 2글자 이상 입력하면, 300ms의 디바운스(debounce) 타임 이후 자동으로 검색을 시작하여 불필요한 API 호출을 최소화한다. 검색 결과는 현재 페이지를 벗어나지 않는 드롭다운 형태로 최대 10개까지 표시되며, '결과 더보기'를 클릭하면 전체 검색 결과 페이지로 이동한다.</li><li><b>검색 로직:</b> 검색 대상은 사용자가 접근 권한을 가진 모든 파일 및 폴더의 이름으로 한정한다. 검색 엔진은 부분 일치(contains) 및 한글 초성 검색(예: 'ㅅㅈ' 입력 시 '사진' 검색)을 지원해야 한다. 검색 결과에는 파일 유형을 나타내는 아이콘, 파일명(검색 키워드는 볼드체로 하이라이트 처리), 파일의 상위 폴더 경로, 마지막 수정일 정보가 포함되어야 한다. 서버에서는 검색 성능을 보장하기 위해 파일 메타데이터에 대한 인덱싱(예: Elasticsearch)을 구축해야 한다.</li></ul>
