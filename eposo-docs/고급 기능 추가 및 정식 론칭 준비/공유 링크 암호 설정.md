# 공유 링크 암호 설정

<h3>Objective</h3><p>사용자가 파일을 공유할 때 암호를 설정하여, 링크를 아는 사람이라도 허가된 인원만 콘텐츠에 접근할 수 있도록 보안을 강화한다.</p><h3>Key Results</h3><ul><li>암호 설정 및 해제 UI 반응 시간 500ms 미만</li><li>암호가 설정된 링크 접근 시, 암호 입력 페이지 로딩 시간 1초 미만</li><li>암호화된 링크의 무단 접근 시도 100% 차단</li></ul><h3>기능 명세</h3><ul><li><b>암호 설정 UI:</b> 파일 공유 설정 모달 내에 '암호 설정' 토글 스위치를 추가한다. 사용자가 토글을 켜면, 암호를 입력할 수 있는 텍스트 필드가 나타난다. 암호는 최소 4자 이상이어야 하며, 입력 시 마스킹 처리된다. 시스템이 '강력한 암호 자동 생성' 버튼을 제공하여 사용자 편의를 돕는다. 암호가 설정되면 공유 링크 정보와 함께 암호가 설정되었음을 나타내는 아이콘이 표시된다.</li><li><b>접근 제어:</b> 암호가 설정된 공유 링크에 접속한 사용자는 파일 내용을 보기 전에 암호를 입력하는 페이지로 먼저 리디렉션된다. 올바른 암호를 입력해야만 파일에 접근할 수 있다. 5회 연속으로 잘못된 암호를 입력할 경우, 해당 IP 주소에서의 접근을 15분간 차단하여 무차별 대입 공격(Brute-force attack)을 방지한다.</li><li><b>보안:</b> 설정된 암호는 서버에 평문으로 저장되지 않고, 솔트(salt)를 추가하여 해싱된 값으로 저장되어야 한다. 모든 인증 과정은 HTTPS를 통해 안전하게 처리된다.</li></ul>
