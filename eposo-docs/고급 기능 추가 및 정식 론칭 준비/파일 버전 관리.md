# 파일 버전 관리

<h3>Objective</h3><p>사용자가 실수로 파일을 덮어쓰거나 잘못 수정한 경우, 이전 상태로 쉽게 복원할 수 있도록 파일의 변경 이력을 자동으로 관리하는 기능을 제공한다.</p><h3>Key Results</h3><ul><li>파일 복원 작업 성공률 100%</li><li>이전 버전 목록 조회 API 응답 시간 500ms 미만</li><li>버전 관리로 인한 추가 스토리지 사용량 원본 파일 크기의 평균 30% 이내로 관리</li></ul><h3>기능 명세</h3><ul><li><b>버전 생성:</b> 사용자가 클라우드에 있는 파일과 동일한 이름의 파일을 업로드할 경우, 기존 파일을 덮어쓰는 대신 이전 버전을 자동으로 저장한다. 파일의 변경 이력은 최대 30일 또는 최근 10개 버전까지 보관된다. 텍스트 기반 파일(.txt, .md, .docx 등)의 경우, 전체 파일을 저장하는 대신 변경된 부분(diff)만 저장하여 스토리지 효율을 높인다.</li><li><b>버전 확인 및 복원:</b> 사용자는 파일의 컨텍스트 메뉴에서 '버전 관리' 또는 '변경 내역' 옵션을 선택할 수 있다. 이를 클릭하면 해당 파일의 모든 버전 목록을 시간 순으로 보여주는 모달 창이 열린다. 각 버전 항목에는 저장된 시간, 수정한 사용자, 파일 크기 정보가 표시된다. 사용자는 특정 버전을 선택하여 '미리보기' 하거나 '이 버전으로 복원' 버튼을 클릭하여 현재 파일을 해당 버전으로 되돌릴 수 있다. 복원 시, 현재 버전은 새로운 이전 버전으로 저장되어 복원 작업 자체도 되돌릴 수 있다.</li></ul>
